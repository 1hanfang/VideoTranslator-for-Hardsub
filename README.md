# 视频硬字幕翻译器

把视频里**内嵌（硬压）的日语/英语字幕**自动识别出来，用**本地大语言模型**翻译成简体中文，再把中文字幕渲染回视频，输出一个新视频文件。

**完全本地运行**：视频内容、识别结果、译文全程不离开你的电脑；除首次运行下载环境和模型外，之后可完全离线使用。

## 功能特性

- 🎯 **OCR 识别硬字幕**：基于 RapidOCR（PP-OCR），支持日文/英文，不挑字幕颜色和字体
- 🤖 **内置本地 LLM 翻译**：[Sakura-GalTransl-7B-v3.7](https://huggingface.co/SakuraLLM/Sakura-GalTransl-7B-v3.7)，为日式视觉小说/动画对话场景专项训练的翻译模型，译文自然口语化，自动维护人名术语表保证全片人名一致，不回避成人向文本
- ✂️ **两种字幕模式**：
  - **替换原字幕**：擦除原字幕（时域背景回填 + 图像修复），在原位置按原字幕的颜色样式重绘中文
  - **原字幕旁加中文**：保留原字幕，中文以描边文字放在原字幕下方（放不下自动放上方），无背景条不遮挡画面
- ⚡ **效率优化**：只对含字幕的画面做处理；同一条字幕全片只识别、翻译一次；NVIDIA 显卡硬件编码（NVENC）
- 📐 **无损规格**：输出视频的分辨率、帧率、时长与原视频一致，音频原样复制
- ✏️ **可选人工校对**：翻译完成后可在网页上直接修改译文，再合成视频
- 💾 **断点续跑**：长视频任务中断后重新开始，自动从已完成的阶段继续；任务成功后自动清理临时缓存

## 使用方法

1. **双击 `启动.bat`**。首次需要以管理员身份打开，运行会自动完成全部环境配置（见下文"首次运行"），之后每次几秒钟即可启动
2. 浏览器自动打开操作页面，**拖入视频或点击选择文件**
3. 选择源语言（自动/日语/英语）和字幕模式，点**开始翻译**
4. 等待四个阶段（扫描 → 识别 → 翻译 → 合成）完成，页面会显示输出文件路径

**输出位置**：输出文件命名为 `原文件名.zh.mp4`。点击选择的视频输出到原视频同目录；拖拽的视频会自动按文件名和大小定位原文件位置并输出到同目录，定位不到时输出到Windows系统"视频\OCRtranslator"文件夹。

## 环境要求

| 项目 | 要求 |
|---|---|
| 操作系统 | Windows 10 / 11（64 位）。程序依赖 Windows 批处理、系统字体和文件对话框，不支持 macOS/Linux |
| 显卡 | **推荐 NVIDIA 显卡、显存 ≥ 6GB**（翻译模型经 Vulkan 全量加载入显存，视频编码走 NVENC）。支持 Vulkan 的 AMD/Intel 显卡理论上也能跑翻译（未充分测试），无 NVENC 时编码自动回退 CPU（libx264，速度变慢）。纯核显/无独显也能运行，但翻译速度会显著下降 |
| 内存 | ≥ 16GB（实测环境 32GB） |
| 磁盘空间 | 约 6GB（Python 环境 + 推理引擎 + 模型），任务运行中另需约 0.5–1GB/小时视频的临时缓存（完成后自动清理） |
| Python | **无需自装**。有系统 Python(≥3.9) 时自动创建独立虚拟环境；没有时自动下载便携版，均不影响系统环境 |
| ffmpeg | **无需自装**，本机没有时自动下载 |
| 网络 | 仅首次运行需要（下载依赖与模型，走国内镜像源）；之后完全离线 |

### 首次运行会下载什么

全部自动完成，总计约 4.5GB，存放在程序目录内，不写注册表、不装系统服务：

| 内容 | 大小 | 来源 |
|---|---|---|
| Python 依赖（FastAPI、OpenCV、RapidOCR 等） | ~300MB | 清华 PyPI 镜像 |
| llama.cpp 推理引擎（Vulkan 版） | ~31MB | GitHub |
| Sakura-GalTransl-7B-v3.7 翻译模型（IQ4_XS 量化） | ~4.2GB | hf-mirror.com（支持断点续传，中断后重新运行继续下载） |
| OCR 检测/识别模型（日/英） | ~30MB | ModelScope |
| ffmpeg（仅本机没有时） | ~80MB | gyan.dev |

## 性能参考

实测环境：RTX 5060 Laptop（8GB 显存）/ 32GB 内存：

- 34 分钟 720p30 视频：端到端约 38 分钟（约为视频时长 1.1 倍）
- 翻译速度约 1.6 秒/条字幕；1080p60 视频合成约 56 帧/秒
- 显存占用峰值约 5GB（翻译阶段）

## 限制与已知问题

- 只处理**硬字幕**（画在画面里的字幕）；外挂/封装字幕轨请用常规字幕工具
- 只检测**画面下方 45% 区域**的文字，画面顶部的注释文字不会被处理
- 翻译模型为**日语 → 简体中文**专项优化；英语视频可用但翻译质量一般
- 逐字打出的"打字机式"字幕会被切成多条；花体/艺术字可能识别失败
- "替换原字幕"模式在画面剧烈运动的区域擦除后可能有轻微模糊痕迹（多数会被新字幕覆盖）；介意的话请用"原字幕旁加中文"模式
- 首次启动翻译引擎约需 40 秒加载模型
- 关闭黑色控制台窗口会停止程序（任务会中断，重开后可断点续跑）
